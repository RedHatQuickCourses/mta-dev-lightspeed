= Lab 1: Deploying Coolstore on JBoss EAP 7

In this lab, you deploy the legacy **Coolstore** application as a containerized workload on **JBoss EAP 7** running on OpenShift. This establishes the baseline environment and allows you to observe how the legacy javax-based application behaves in its current, production-like state.

You follow predefined steps to build and deploy the container image, verify the application functionality, and confirm that Coolstore is running correctly on EAP 7.

== Instructions

* _Access the Red Hat OpenShift Cluster_

1. Log in to the OpenShift cluster provisioned for this lab using the administrator credentials provided.
+
[source,subs="verbatim,quotes",bash]
----
**oc login -u admin -p <password> https://api.cluster-<GUID>.dynamic.redhatworkshops.io:6443**
----

* _Prepare and Build the Legacy Application_

1. Clone the Git repository containing the legacy application that will be used throughout this training.
+
[source,subs="verbatim,quotes",bash]
----
**git clone https://github.com/opentraining-labs-customizations/getting-started.git**
----

2. Navigate to the directory containing the cloned source code.
+
[source,subs="verbatim,quotes",bash]
----
**cd getting-started**
----

3. Update the Keycloak configuration to match your Red Hat OpenShift environment. Edit the following file and replace the placeholder <GUID> with your OpenShift cluster identifier.
+
[source,subs="verbatim,quotes",bash]
----
**vi coolstore/src/main/webapp/keycloak.json**
{
    "realm": "eap",
    "auth-server-url": "https://keycloak-coolstore-mono.apps.cluster-**<GUID>**.dynamic.redhatworkshops.io/",
    "ssl-required": "external",
    "resource": "eap-app",
    "public-client": true,
    "confidential-port": 0
}
----

4. Build the application using Maven so that the updated configuration is included in the deployable artifact.
+
[source,subs="verbatim,quotes",bash]
----
**mvn clean package -pl coolstore -am**
----

5. Verify that the application archive has been successfully generated.
+
[source,subs="verbatim,quotes",bash]
----
**ls coolstore/target/ROOT.war**
----

* _Deploy the Legacy Coolstore Application on OpenShift (JBoss EAP 7)_

1. Clone the repository that contains the OpenShift resources and deployment configuration for the legacy application.
+
[source,subs="verbatim,quotes",bash]
----
**git clone https://github.com/opentraining-labs-customizations/mta-dev-lightspeed.git**
----

2. Copy the previously built application WAR file into the deployment repository.
+
[source,subs="verbatim,quotes",bash]
----
**ls**
getting-started  mta-dev-lightspeed

**cp getting-started/coolstore/target/ROOT.war \**
**mta-dev-lightspeed/eap7_coolstore/00-eap7-coolstore-image **
----

3. Navigate to the image build directory and create the required BuildConfig and ImageStream for JBoss EAP 7.
+
[source,subs="verbatim,quotes",bash]
----
**cd mta-dev-lightspeed/eap7_coolstore/00-eap7-coolstore-image/**

**oc create -f 00-bc-is.yaml**
namespace/coolstore-mono created
buildconfig.build.openshift.io/eap74-coolstore created
imagestream.image.openshift.io/eap74-coolstore created
----
+
Verify that the resources have been created successfully.
+
[source,subs="verbatim,quotes",bash]
----
**oc project coolstore-mono**
Now using project "coolstore-mono" on server "https://api.cluster-6f2l2.dynamic.redhatworkshops.io:6443".

**oc get bc**
NAME              TYPE     FROM     LATEST
eap74-coolstore   Docker   Binary   0

**oc get is**
NAME              IMAGE REPOSITORY                                                                  TAGS   UPDATED
eap74-coolstore   image-registry.openshift-image-registry.svc:5000/coolstore-mono/eap74-coolstore
----

4. Start a binary build using the copied application artifact.
+
[source,subs="verbatim,quotes",bash]
----
**oc start-build eap74-coolstore --from-dir=. --follow**
----

5. Navigate back to the main deployment directory and deploy the PostgreSQL database along with its service.
+
[source,subs="verbatim,quotes",bash]
----
**cd mta-dev-lightspeed/eap7_coolstore**

**oc create -f 01-postgresql-deploy-svc.yaml**
deployment.apps/postgres created
service/postgres created
----

6. Verify that the database deployment and service are running.
+
[source,subs="verbatim,quotes",bash]
----
**oc get deployment**
NAME       READY   UP-TO-DATE   AVAILABLE   AGE
postgres   1/1     1            1           2m21s

**oc get svc**
NAME       TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)    AGE
postgres   ClusterIP   172.31.185.74   <none>        5432/TCP   2m32s
----

* _Configure and Deploy Keycloak_

1. Create a ConfigMap containing the Keycloak realm configuration.
+
[source,subs="verbatim,quotes",bash]
----
**oc create configmap keycloak-realm --from-file=realm-export.json**
configmap/keycloak-realm created

**oc get configmap keycloak-realm**
NAME             DATA   AGE
keycloak-realm   1      20s
----

2. Edit the Keycloak deployment file and replace <GUID> with your OpenShift cluster identifier.
+
[source,subs="verbatim,quotes",bash]
----
** vi 02-keyclaok-deploy-svc-route.yaml**
...
            - name: KC_PROXY
              value: edge
            - name: KC_HOSTNAME
              value: keycloak-coolstore-mono.apps.cluster-<GUID>.dynamic.redhatworkshops.io
            - name: KC_HOSTNAME_STRICT
              value: "false"
            - name: KC_HTTP_ENABLED
              value: "true"
...
----

3. Deploy Keycloak and confirm that all associated resources are created successfully.
+
[source,subs="verbatim,quotes",bash]
----
**oc create -f 02-keyclaok-deploy-svc-route.yaml**
imagestream.image.openshift.io/keycloak-rhbk created
deployment.apps/keycloak created
service/keycloak created
route.route.openshift.io/keycloak created
job.batch/keycloak-bootstrap created

**oc get deployment keycloak**

4. Create a job to provision a default Keycloak user (user1 / redhat) required by the application.

[source,subs="verbatim,quotes",bash]

* _Deploy the Coolstore Application_

1. Create the ConfigMap required for JMS configuration.
+
[source,subs="verbatim,quotes",bash]
----
**oc create configmap eap-jms-cli --from-file=jms.cli**
configmap/eap-jms-cli created
----

2. Deploy the Coolstore application along with its service and route.
+
[source,subs="verbatim,quotes",bash]
----
**oc create -f 04-coolstore-deploy.yaml**
deployment.apps/coolstore-app created
service/coolstore created
route.route.openshift.io/coolstore created
----

3. Verify that the application has been deployed successfully and is accessible.
+
[source,subs="verbatim,quotes",bash]
----
oc get deployment/coolstore-app

oc get svc/coolstore

oc get route/coolstore
